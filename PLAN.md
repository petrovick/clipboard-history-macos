# OptionVClipboard: Private macOS Clipboard History

## Goal

Build a private native macOS clipboard history app for personal use.

The app should:

- Run locally on the user's Mac.
- Use no third-party runtime software.
- Watch normal macOS copy operations, including `Command + C`.
- Open clipboard history with `Option + V`.
- Store clipboard history encrypted on disk.
- Keep the encryption key in macOS Keychain.
- Default to safer privacy behavior.

The recommended first version does not replace `Command + V`. Normal paste remains unchanged.

## Core User Flow

1. User copies text normally with `Command + C`.
2. The source app writes the copied text to the macOS pasteboard.
3. OptionVClipboard notices that the pasteboard changed.
4. The app reads the new plain-text clipboard value.
5. The app filters unsafe/sensitive-looking values.
6. The app adds allowed values to encrypted local history.
7. User presses `Option + V`.
8. A floating history picker opens.
9. User selects an item.
10. The selected item becomes the current clipboard value.
11. User presses normal `Command + V` to paste it.

This means the app does not need to intercept `Command + C`. It monitors the system pasteboard after copy operations happen.

## Important v1 Decisions

- Shortcut: `Option + V`.
- Normal paste: keep `Command + V` unchanged.
- Content type: plain text only.
- Auto-paste: not included in v1.
- History storage: encrypted file.
- Encryption key: macOS Keychain.
- UI style: small menu-bar app plus floating picker.
- Dependencies: Apple frameworks only.
- Install target: `~/Applications/OptionVClipboard.app`.

Auto-paste can be added later, but it would likely require Accessibility permission because the app would need to simulate keyboard input into another app.

## Technical Stack

- Language: Swift.
- Build system: Swift Package Manager.
- UI: AppKit.
- Clipboard API: `NSPasteboard`.
- Global shortcut: first try native hotkey registration using Apple APIs; use lower-level event handling only if necessary.
- Encryption: CryptoKit `AES.GCM`.
- Key storage: macOS Keychain through Security framework.
- Startup option: `SMAppService` or a user LaunchAgent if needed.

No Electron, no Homebrew package, no third-party clipboard library, and no network service.

## Component Plan

### 1. App Shell

Build a small AppKit app that runs in the menu bar.

Requirements:

- No normal dock icon by default.
- Menu-bar icon is visible.
- Menu items:
  - Open History
  - Pause Capture / Resume Capture
  - Clear History
  - Open Storage Folder
  - Quit

Acceptance checks:

- App launches.
- Menu-bar icon appears.
- App can quit cleanly.
- No main window appears unless history is opened.

### 2. Clipboard Watcher

Use `NSPasteboard.general.changeCount` to detect clipboard changes.

Requirements:

- Poll pasteboard every 0.5 to 1.0 seconds.
- Before reading or storing any text, inspect the pasteboard item's types and skip the change if it includes a do-not-retain marker.
- Read plain-text content only.
- Ignore empty values.
- Ignore exact duplicates.
- Ignore values larger than the configured max size.
- Ignore clipboard writes made by OptionVClipboard itself.
- Do not capture while paused.

Do-not-retain pasteboard markers:

```text
org.nspasteboard.ConcealedType
org.nspasteboard.TransientType
org.nspasteboard.AutoGeneratedType
com.apple.is-sensitive
```

These markers must be checked before regex-based secret filtering. Password managers and security-sensitive apps may use marker types to signal that clipboard contents should not be retained. A text-only regex filter is not enough.

Own-write guard:

- Keep a small in-memory set of ignored pasteboard change counts.
- When OptionVClipboard writes a selected history item to `NSPasteboard`, read `NSPasteboard.general.changeCount` after the write completes and add that value to the ignored set.
- On the watcher poll, if the current `changeCount` is in the ignored set, remove it from the set and skip processing that change.
- Do not rely on `setString` returning a change count; `setString` returns success/failure. Read `changeCount` from the pasteboard after the write.

Default settings:

```text
Max history items: 100
Max item size: 100 KB
Retention period: 7 days
Content type: text only
```

Acceptance checks:

- Copying normal text adds it to history.
- Copying the same text repeatedly does not create duplicate spam.
- Clipboard items containing do-not-retain marker types are skipped before their text is stored.
- Selecting an existing history item does not cause the watcher to re-ingest the app's own clipboard write.
- Pausing capture prevents new copies from being stored.
- The app must be running to capture new copies.

### 3. History Model

Use a simple item model.

Suggested fields:

```text
id: UUID
text: String
createdAt: Date
source: optional String, if discoverable without extra permissions
```

Rules:

- Newest items appear first.
- Exact duplicates move existing item to the top instead of creating a second copy.
- Old items are removed based on retention limit.
- Item count is capped.

### 4. Secret-Safety Filtering

Add conservative filters before storing clipboard text.

First, skip pasteboard items that include do-not-retain marker types:

```text
org.nspasteboard.ConcealedType
org.nspasteboard.TransientType
org.nspasteboard.AutoGeneratedType
com.apple.is-sensitive
```

Then apply text-pattern filters.

Skip values that look like:

- Private keys.
- Long random API tokens.
- Strings containing obvious secret labels such as:
  - `password=`
  - `token=`
  - `secret=`
  - `api_key=`
  - `authorization:`
- Very short one-time codes, if the user prefers.

Important limitation:

No local clipboard app can perfectly know whether copied text is a password or secret. Encryption protects the saved file, but the app must see clipboard text while it is running. The safest defaults are text-only history, respect for pasteboard do-not-retain markers, short retention, secret filtering, and a visible pause button.

### 5. Encrypted Storage

Store history at:

```text
~/Library/Application Support/OptionVClipboard/history.enc
```

Requirements:

- Generate a random symmetric key on first launch.
- Store that key in macOS Keychain.
- Keep plaintext history only in memory while the app is running.
- Serialize history to JSON in memory.
- Encrypt JSON using CryptoKit `AES.GCM`.
- Save only encrypted bytes to disk.
- Use atomic writes:
  - write encrypted bytes to a temporary file
  - replace the old history file

Acceptance checks:

- Restarting the app reloads history.
- Opening `history.enc` does not reveal copied text.
- No plaintext history file exists in Application Support.
- If the Keychain key is deleted, existing encrypted history is expected to become unreadable.

### 6. Keychain Storage

Use macOS Keychain for the encryption key.

Requirements:

- Create key once on first run.
- Reuse existing key on later launches.
- Use a stable service/account name, for example:

```text
Service: OptionVClipboard
Account: history-encryption-key
```

Acceptance checks:

- Key is not stored in the project folder.
- Key is not stored next to the encrypted history file.
- App can decrypt history after restart.

### 7. Global Shortcut

Use `Option + V` to open the picker.

Requirements:

- Register a global hotkey for `Option + V`.
- If registration fails, show a clear error.
- Keep `Command + V` untouched.
- Later allow changing shortcut if `Option + V` conflicts with keyboard layout or another app.

Implementation note:

First try native macOS hotkey registration. If this does not work reliably on the target macOS version, evaluate a fallback using event handling. This must be verified during implementation on the target Mac.

Acceptance checks:

- Pressing `Option + V` from another app opens history.
- Normal `Command + V` still works.
- Normal `Command + C` still works.

### 8. History Picker UI

Build a floating picker window.

Requirements:

- Opens when user presses `Option + V`.
- Shows newest history items first.
- Includes search input.
- Supports keyboard navigation:
  - Up/down arrows move selection.
  - Enter copies selected item to clipboard and closes picker.
  - Escape closes picker.
- Clicking an item copies it to clipboard and closes picker.

v1 behavior:

- Selecting an item copies it to the clipboard.
- User manually presses `Command + V` to paste.
- The app records its own pasteboard write's `changeCount` after copying the selected item, so the watcher skips that internal write.

Acceptance checks:

- `Option + V` opens picker.
- Search filters history.
- Enter selects current item.
- Selected item becomes the current clipboard value.
- Selecting an item does not duplicate or reorder history through the clipboard watcher unless the app explicitly chooses to do so outside the watcher path.
- User can paste selected item with normal `Command + V`.

## Known Risks and Open Checks

### Shortcut Behavior

`Option + V` is safer than changing `Command + V`, but it may conflict with some keyboard layouts or apps. This must be tested on the target Mac. If it conflicts, use `Control + Option + V` as the fallback shortcut.

### Permissions

The recommended v1 copies the selected history item back to the clipboard and does not auto-paste. This should reduce the need for Accessibility permission.

If a later version adds automatic paste into the previous app, macOS will likely require Accessibility permission because the app must synthesize keyboard input.

### Clipboard Privacy

The app has access to copied text while it is running. This is unavoidable for clipboard history. Encryption protects history at rest, not text currently in memory or text currently on the system clipboard. The implementation must respect pasteboard do-not-retain markers before storing or filtering text.

### Secure Fields

Some apps may intentionally avoid placing sensitive field contents on the pasteboard, but if an app does put text on the pasteboard, this app may see it unless paused or filtered.

- The app does not re-ingest its own clipboard writes.
